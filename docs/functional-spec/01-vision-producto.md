# Vision de Producto

## Contexto

El producto nace para operar un negocio real (no template multiproposito). La pagina publica capta interesados y el backend permite gestionar contactos de forma ordenada y trazable.

## Problema a resolver

- Los mensajes de potenciales clientes se pierden entre redes sociales y mensajeria informal.
- No hay trazabilidad clara del estado de cada contacto.
- No existe una frontera formal entre experiencia publica y operacion interna.

## Propuesta de valor

- Experiencia publica clara para mostrar servicios y convertir visitas en solicitudes.
- Backoffice simple para administrar mensajes y seguimiento.
- Flujo confiable de captura, registro y gestion de contactos.

## Objetivos de negocio

- Aumentar conversion de visitante a contacto.
- Reducir tiempo de respuesta inicial.
- Asegurar que ninguna solicitud valida se pierda.

## Principios del producto

- Simplicidad funcional sobre complejidad.
- Seguridad por defecto.
- Trazabilidad y consistencia de datos.
- Evolucion incremental por fases.
